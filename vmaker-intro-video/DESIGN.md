# vMaker Intro Video Design

## Style Prompt

Dark cinematic project-index film for vMaker, based on the existing site: black-to-near-black canvas, a soft pink accent, glassy panels, subtle radial light, thin borders, compact data labels, and a continuous movement from hero identity into structured repository intelligence. The video should feel like a polished product system walkthrough, not a generic portfolio promo.

## Colors

- Background: `#050505`
- Panel: `#101014`
- Foreground: `#F8F7FA`
- Muted text: `#A9A2AE`
- Accent pink: `#F598F2`
- Secondary cyan: `#58D5FF`
- Data lime: `#A3E635`

## Typography

- Primary: `Figtree`, matching the app's global font.
- Data/code voice: `JetBrains Mono`.
- Large display type uses medium weight; small labels use mono or uppercase treatment without extra letter spacing.

## Motion

- Confident, medium-energy product motion.
- Use push and mask-style transitions, with one consistent horizontal wipe between scenes.
- Entrances should stagger by meaning: product name, source of truth, data pipeline, browsing experience, closing thesis.
- Ambient motion stays slow and finite: drifting glow, scan line movement, panel parallax.

## What NOT to Do

- Do not add a second visual brand or generic blue-purple SaaS gradient.
- Do not use stock footage, screenshots, or remote API calls.
- Do not make jump cuts between scenes.
- Do not overload the frame with unreadable repository details.
- Do not imply there is a backend database; the project intentionally uses GitHub as source of truth.
