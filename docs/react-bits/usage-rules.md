# vMaker React Bits 使用规则

来源：`components.json` 中配置的 React Bits registry

## 角色定位

React Bits 用于给简约作品集站提供可复用的视觉效果组件。它不负责决定页面布局、内容策略或基础组件系统。

## 适配优先规则

选择 React Bits 组件时，优先看它是否最适合当前交互和视觉目标。已有本地组件可以复用，但如果 registry 中有更适合的组件，即使本地已有类似组件，也可以重新安装并使用更合适的组件。

决策顺序：

1. 先定义清楚视觉任务：文字揭示、媒体 hover、背景点缀、空状态、项目预览动效等。
2. 检查已有本地 React Bits 组件是否能干净地完成这个任务。
3. 如果 registry 里有更合适的组件，就安装并使用那个组件。
4. 对选中的组件做轻量封装或适配，让它符合项目 props 和布局需求。
5. 只有当 React Bits 无法覆盖需求时，才写自定义效果。

## 适合使用的位置

- Hero 标题或简短介绍文字的入场揭示。
- 精选项目预览的入场动效。
- 项目媒体区域的小型 hover 反馈。
- 空状态或加载状态的视觉打磨。

## 不适合使用的位置

- 基础按钮、输入框、链接、弹窗或导航组件。
- 同时给每一张项目卡片都加重动效。
- 无限循环的装饰背景。
- 会在动画完成前隐藏关键信息的效果。

## 安装规则

当最合适的组件需要新增或刷新时，使用已配置的 registry，不要手动复制粘贴：

```bash
npx shadcn@latest add @react-bits/<component-name>
```

安装后的视觉效果组件建议放在：

```text
app/components/react-bits/
```

## 动效约束

- 交互动效默认 150-300ms，页面介绍类动效应控制在 700ms 以内。
- 必须尊重 `prefers-reduced-motion`。
- 避免布局位移。优先动画 opacity、transform、filter 或 clip，并保持尺寸稳定。
- 不使用持续循环动画，除非它表达加载或进度。

## 审查清单

- 关闭所有动效后，页面仍然有用。
- 动效服务于明确的信息层级。
- 动效不会和项目截图争夺注意力。
- 不为了很小的视觉收益引入沉重依赖。
- 移动端不会出现文字重叠或布局错位。
