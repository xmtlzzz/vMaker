# vMaker 产品上下文

## 产品定位

vMaker 是一个个人项目展示网站，用来集中展示所有个人项目。它需要帮助访问者快速理解：我做过什么、每个项目为什么值得看、以及如何打开、查看或进一步了解这些项目。

## 页面属性

这是一个品牌导向的个人作品集网站。设计本身是项目气质的一部分，但它不能变成纯装饰画廊；它仍然要像一个清晰的项目索引，让访问者能快速浏览和比较项目。

## 目标用户

- 招聘方、潜在合作者、技术同行：他们会用这个站点评估我的能力、项目质量和技术方向。
- 普通访问者或朋友：他们需要用较低理解成本看懂这些项目大概是什么。
- 我自己：后续会持续新增项目，因此结构必须易维护、易扩展。

## 核心任务

- 用足够清晰的上下文展示项目，让访问者能判断项目质量、范围和相关性。
- 让访问者可以顺畅地筛选、浏览、打开项目。
- 通过简洁描述、真实截图、状态标签和明确链接建立可信度。
- 使用可复用的项目数据结构和组件模式，方便后续继续添加项目。

## 第一版数据源

第一版不引入 Neon、后台管理页或登录系统。项目事实数据直接来自 GitHub 用户 `xmtlzzz` 的公开仓库。

GitHub 负责提供：

- 仓库名称。
- 仓库地址。
- 项目描述。
- Homepage/Demo 链接。
- 主要语言。
- Topics。
- star、fork 等基础指标。
- 最近更新时间和 push 时间。
- 最近 commit 记录。
- languages 接口返回的语言字节分布。

vMaker 只负责展示、筛选、统计和视觉包装。需要自定义展示名、封面、排序、精选或隐藏项目时，先使用本地配置文件处理，不引入数据库。

注意：GitHub languages 接口返回的是各语言的字节数，不是精确代码行数。第一版展示“语言构成”和“代码体量”，不展示“精确代码行数”。

## 品牌气质

整体气质应当是简约、现代、克制、有技术感，但不能冷淡或模板化。网站应像一个经过策展的个人工作室索引：界面安静、排版精确、动效克制，主要视觉个性由项目截图、预览和项目内容本身承担。

## 设计原则

- 清晰优先于装饰。每一个视觉元素都必须帮助访问者理解或比较项目。
- 使用克制的中性色基础，再配一个明确、有记忆点的强调色。
- 让项目截图、缩略图或产品预览承担主要视觉丰富度。
- 项目信息要紧凑但可读，不要把页面做成空洞的大型营销页。
- 避免模板化作品集套路：泛用渐变光斑、重复图标卡片、虚假仪表盘、含糊的功能文案。

## 部署背景

网站后续会部署到 Vercel。UI 应当快速、响应式良好，并适合静态化部署。除非动效能明显改善项目浏览体验，否则不要引入沉重的运行时效果。
